## #8 — Flow đề xuất (PLAN-ONLY, chờ user review)

### Bối cảnh
Khi PM/điều phối sửa root task (parent = null), thường liên quan đến cam kết chính của dự án nên cần:
- Phân biệt **thay đổi Timeline** (baselineStartDate / baselineEndDate / startDate / endDate) vs **thay đổi Giờ công** (plannedHours)
- Mỗi loại thay đổi yêu cầu **file đã được phê duyệt** đính kèm làm bằng chứng

### Flow đề xuất

```
┌────────────────────────────────────────────────────────────────────┐
│ User mở Edit-modal cho root task                                   │
│   parentId === null                                                │
└──────────────────────────────┬─────────────────────────────────────┘
                               ▼
┌────────────────────────────────────────────────────────────────────┐
│ Modal chia làm 3 nhóm                                              │
│   ① Metadata (name, deliverable, ownerId...) — không cần BC        │
│   ② Timeline   — baseline + actual start/end                        │
│   ③ Giờ công   — plannedHours                                       │
│                                                                    │
│ ② và ③ mỗi nhóm có:                                                │
│   • So sánh giá trị cũ ↔ mới (text-diff inline)                    │
│   • Field "Tải bằng chứng" (file uploader)                         │
│   • Field "Ghi chú thay đổi" (textarea bắt buộc)                   │
└──────────────────────────────┬─────────────────────────────────────┘
                               ▼
┌────────────────────────────────────────────────────────────────────┐
│ Validate khi Save:                                                 │
│   IF nhóm ② có dirty fields → bắt buộc evidenceFile + note         │
│   IF nhóm ③ có dirty fields → bắt buộc evidenceFile + note         │
│   IF cả 2 → bắt buộc 2 file riêng (HOẶC dùng chung 1 file)         │
│   ELSE → save bình thường, không cần evidence                       │
└──────────────────────────────┬─────────────────────────────────────┘
                               ▼
┌────────────────────────────────────────────────────────────────────┐
│ BE side:                                                           │
│   • Evidence file → projectDocuments với category mới              │
│     'TASK_CHANGE_EVIDENCE' (timeline) / 'TASK_HOURS_EVIDENCE'      │
│   • Tham chiếu doc trong ActivityLog (entityId = planItemId)       │
│   • Activity action mới:                                           │
│       TASK_TIMELINE_CHANGED  — có evidenceDocId                    │
│       TASK_HOURS_CHANGED     — đã tồn tại, bổ sung evidenceDocId   │
└──────────────────────────────┬─────────────────────────────────────┘
                               ▼
┌────────────────────────────────────────────────────────────────────┐
│ FE Activity Log:                                                   │
│   Mỗi entry có link "Xem bằng chứng" → download file               │
└────────────────────────────────────────────────────────────────────┘
```

### #7 — Worklog approval (chờ Q3)
- Schema:
  ```prisma
  enum WorklogStatus { PENDING APPROVED REJECTED }
  model Worklog {
    ...
    status         WorklogStatus @default(PENDING)
    decidedById    String?
    decidedBy      User?         @relation("WorklogDecided", fields: [decidedById], references: [id])
    decidedAt      DateTime?
    rejectReason   String?
  }
  ```
- Endpoint mới: `PATCH /api/projects/:projectId/worklogs/:wlId/decide` body `{decision, reason?}`
- Permission: chỉ PM dự án + điều phối + PMO mới được decide; không cho self-decide worklog của chính mình
- Recalc `task.actualHours`: chỉ tính worklog APPROVED
- Activity log: 2 action mới `WORKLOG_APPROVED`, `WORKLOG_REJECTED`
- FE: Modal "Chi tiết worklog" mới — hiển thị thông tin worklog + nút Duyệt/Từ chối (kèm reason input)
- Member workspace: hiển thị status worklog của user

### #9 — Subtask permissions + scope
