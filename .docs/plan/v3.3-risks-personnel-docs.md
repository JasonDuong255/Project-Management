# v3.3 — Risks expanded, KH/Đối tác catalog, document upload

**Goal.** Bring risks, personnel, and documents up to BRD spec. Adds Supabase Storage for the first time.

**Depends on.** v3.1 (schema base).

## Tasks

### Risks (audit IV.5.1.4)

- [ ] **Schema**: add columns to `project_risks`:
  - `cause` text, `description` text, `dueDate` date, `resolutionResult` text, `resolutionProgress` int (0–100), `nextPlan` text, `notes` text.
- [ ] **DELETE endpoint**: `DELETE /api/projects/:id/risks/:riskId` — gated by `canManageProjectPlan`.
- [ ] **History**: log `RISK_CREATED`, `RISK_UPDATED`, `RISK_DELETED` in `activity_logs`. Add to enum.
- [ ] **FE risk modal**: rich form with all the new fields. Show progress bar based on `resolutionProgress`. Add delete button.

### KH/Đối tác catalog (audit IV.2.7, IV.5.1.2.2)

- [ ] **New tables**: `external_personnel` + `project_external_personnel` (schema in `data-model-delta.md`).
- [ ] **Backfill**: scan all `projects.personnelInfo.customerMembers[]` and `partners[]`; create `external_personnel` rows; link via `project_external_personnel`. **De-dupe** on email.
- [ ] **Endpoints**:
  - `GET /api/external-personnel?kind=CUSTOMER|PARTNER` — list + filter.
  - `POST /api/external-personnel` — create new.
  - `PATCH /api/external-personnel/:id` — update.
  - `POST /api/projects/:id/external-personnel` — link existing or create-and-link.
  - `DELETE /api/projects/:id/external-personnel/:externalPersonnelId` — unlink (preserve history).
- [ ] **Activity log**: `PERSONNEL_ADDED`, `PERSONNEL_REMOVED`.
- [ ] **FE**: ProjectDetailPage personnel tab — picker that searches the catalog OR creates inline. Update sync to global catalog when editing.
- [ ] **AdminCatalogPage**: add a "Nhân sự KH/Đối tác" tab using the same UI pattern.

### Documents upload (audit IV.5.1.3)

- [ ] **Supabase Storage**: create bucket `qlda-documents` (via Supabase Studio or admin API in seed).
- [ ] **Schema**: add `storagePath` column to `project_documents` (kept alongside `url` for backward compat). Add `mimeType`, `sizeBytes`.
- [ ] **Endpoints**:
  - Replace `POST /api/projects/:id/documents` to accept multipart/form-data → upload to bucket → store path + metadata. Or keep a 2-step pattern: presigned URL (`POST /documents/presigned-upload`) then PUT to Supabase, then `POST /api/projects/:id/documents` with the resulting path.
  - `GET /api/projects/:id/documents/:docId/download` — generate a presigned URL with short TTL, return URL.
- [ ] **FE**: replace the "URL" text input with a file dropzone (use `react-dropzone` or native `<input type="file">`). Show progress during upload. Replace "open" links with download (call download endpoint, navigate browser to the resulting URL).
- [ ] **Activity log**: split `DOCUMENT_ADDED` from new `DOCUMENT_UPDATED`. Existing log entries stay as-is.
