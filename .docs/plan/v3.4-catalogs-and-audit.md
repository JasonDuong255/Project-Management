# v3.4 — Catalogs revamp + activity log expansion

**Goal.** Move catalog values from a `Catalogs.json[]` blob to per-row records with audit trails; add the missing catalogs (`ttkForms`, `deploymentForms`); expand activity-log coverage.

**Depends on.** v3.1 (status/health enum changes already shipped).

## Tasks

### Catalog schema redesign

Today: `catalog_groups { key: enum, values: Json }` — values are an opaque array, no per-row audit.

Replace with:

```prisma
model CatalogOption {
  id           String       @id @default(cuid())
  groupKey     CatalogKey
  value        String
  label        String
  description  String       @default("")
  responsibility String     @default("")  // for projectMemberRoles ("nhiệm vụ")
  isActive     Boolean      @default(true)
  isSystem     Boolean      @default(false)  // true → cannot edit/delete (e.g., projectStatuses values)
  createdById  String       @db.Uuid
  createdAt    DateTime     @default(now())
  updatedById  String?      @db.Uuid
  updatedAt    DateTime     @updatedAt
  @@unique([groupKey, value])
}

enum CatalogKey {
  projectStatuses healthStatuses taskStatuses riskLevels
  documentCategories departments projectMemberRoles
  ttkForms deploymentForms     // NEW
}
```

- [ ] Drop the old `catalog_groups` table after migrating its values into `catalog_options`.
- [ ] **Backfill**: each `(groupKey, value, label, description)` becomes a row. Mark `projectStatuses` and `healthStatuses` rows as `isSystem=true`.

### New catalog endpoints

- [ ] `GET /api/catalogs/:groupKey` — list rows (filter by `isActive=true` for end users; admin sees all).
- [ ] `POST /api/catalogs/:groupKey` — create row (rejected with 400 if `isSystem` group).
- [ ] `PATCH /api/catalogs/options/:id` — update row.
- [ ] `DELETE /api/catalogs/options/:id` — soft delete (`isActive=false`) — keeps referential integrity for historical projects.
- [ ] All audit fields auto-populated.

### FE redesign

  - 9 tabs (one per CatalogKey).

  - Editable groups: show CRUD UI with audit columns (createdBy/createdAt/updatedBy/updatedAt). Add "Active" toggle.
