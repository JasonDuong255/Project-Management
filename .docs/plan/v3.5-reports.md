# v3.5 — Reports export

**Goal.** Implement BRD IV.5.5 — exportable reports using customer-supplied templates.

**Depends on.** v3.1 (status enum). Independent of v3.2/v3.3/v3.4 if needed.

## Tasks

- [ ] **Library**: add `exceljs` to `backend/package.json` (more flexible than `xlsx` for templates with formatting).
- [ ] **Templates**: place customer-supplied .xlsx templates in `backend/templates/` and commit. `weekly-report.xlsx` and `detailed-plan.xlsx`.
- [ ] **Endpoints**:
  - `GET /api/reports/weekly?projectId=X&isoWeek=YYYY-WXX` → streams a generated .xlsx.
  - `GET /api/reports/detailed-plan?projectId=X` → streams a .xlsx.
- [ ] **Service** (`backend/src/modules/reports/reports.service.ts`):
  - Load template, fill cells with project data + plan items + worklogs (filtered to the requested week if applicable).
  - For weekly: per-member rows with this week's worklogs grouped by task, plus next-week plan.
- [ ] **FE**: download buttons:
  - On `ReportsPage`: row-level "Tải báo cáo tuần" + "Tải kế hoạch chi tiết" per project.
  - On `ProjectDetailPage`: top-of-page download buttons (PM/QLDA only).
- [ ] **Activity log**: optional `REPORT_EXPORTED`.
